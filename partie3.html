Questions de compréhension

  1. Dans quel cas est-il nécessaire d'inclure l'attribut « enctype="multipart/form-data" » dans la balise < form > d'un formulaire?

    • Lors de l'envoi de fichier via un formulaire .POST, les formulaires .GET ne pouvant traiter de fichiers.

  2. Examinez le code HTML ci-dessous et répondez aux questions suivantes :



    1. À quoi peuvent bien servir les attributs for dans les balises < label >?
    2. Quel est le critère obligatoire à satisfaire pour pouvoir insérer un attribut for dans une balise < label >?
      • Qu'il existe un ID pour répondre à sa demande.
    3. À quoi peuvent bien servir les attributs name dans le balises < input />?
      • Pour pouvoir les cibler en Javascript ou en PHP et alors créer un site web dynamique où les utilisateurs intéragissent au travers d'un formulaire, par exemple.
    4. Pourquoi les attributs name dans les balises < input /> présentent-ils la même valeur, soit « checkbox_radio »?
      • Car ils nous obligent à ne sélectionner qu'une seule et unique réponse. Quand on change de checkbox, le crochet se déplace. Il ne fait pas qu'en ajouter un autre dans la case à cocher. Même chose pour les "boutons" radio.

  3. Quelle est la principale différence entre la méthode « get » et la méthode « post » pour l'envoi d'un formulaire?
    (< form method="get" > comparativement à < form method="post" >)?

    • La méthode get transmet les données directement au travers l'URL, aucune sécurité et étant très limitée, tandis que la méthode post transmet le formulaire au serveur via des requêtes http ou https. Les paramètres ne sont donc pas visibles par les utilisateurs et leur portée est illimité.
      Beaucoup plus sécuritaire.

  4. À partir de la version 5 du HTML (HTML5), plusieurs balises structurantes autre que seulement < div > ou < span > ont été définies (par exemple < main >, < section >, < article >). Pourquoi est-il important d'utiliser les nouvelles balises structurisantes plutôt que d'avoir toujours recours aux balises < div > ou < span > comme c'était le cas avec HTML4 et ses versions antérieures?

    • Pour aider les utilisateurs et les machines à analyser et à repérer la page, par exemple pour les gens aveugles qui utilisent un lecteur d'écran pour naviger en ligne et pour les robots des moteurs de recherche qui font du référencement et qui place ton site au sommet des recherches.
